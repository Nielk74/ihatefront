Light theme only

Build clean product UIs with plain HTML

`ihatefront` ships as CSS plus a small vanilla JavaScript module. Use the CSS for static components, then opt into JS behavior with `data-ih-*` attributes for menus, tabs, accordions, modals, drawers, autocomplete, dismissible alerts, and snackbars.

GitHub Browse components v0.1.0

Install from the repo

npm install ihatefront

Use compiled assets

<link rel="stylesheet" href="dist/ihatefront.css">
<script type="module" src="dist/ihatefront.js"></script>

Foundations

Tokens and global conventions

Customize the framework by overriding CSS variables. The first release supports a light color scheme only.

Variants
plain outlined soft solid
Semantic colors
success warning danger neutral
Density Small controls use `--sm`, large controls use `--lg`, and tables can opt into `--dense`. 
:root {
  --ih-primary: #0969da;
  --ih-radius-sm: 6px;
  --ih-container: 1180px;
}

Use tokens for product-level customization. Avoid writing one-off colors into component markup.

Inputs

Button

Buttons are action controls. Combine `.ih-btn` with one variant, one color, and optionally one size. Motion uses elevation changes and ripple-like press feedback.

Variants

<button class="ih-btn ih-btn--solid ih-btn--primary">Solid</button>
<button class="ih-btn ih-btn--soft ih-btn--primary">Soft</button>
<button class="ih-btn ih-btn--outlined ih-btn--primary">Outlined</button>
<button class="ih-btn ih-btn--plain ih-btn--primary">Plain</button>

Colors and sizes

<button class="ih-btn ih-btn--sm ih-btn--solid ih-btn--neutral">Small</button>
<button class="ih-btn ih-btn--outlined ih-btn--danger">Danger</button>
<button class="ih-btn ih-icon-btn ih-btn--outlined ih-btn--neutral" aria-label="Settings">...</button>
i
Accessibility

Use `button` for actions and `a.ih-btn` for navigation. Icon-only buttons need an `aria-label`.

Use button groups for closely related actions. Add `role="group"` and a label.

  <button class="ih-btn ih-btn--outlined ih-btn--neutral ih-btn--sm">Day</button>
  <button class="ih-btn ih-btn--outlined ih-btn--neutral ih-btn--sm">Week</button>
  <button class="ih-btn ih-btn--outlined ih-btn--neutral ih-btn--sm">Month</button>

Inputs

Toggle group

Use toggle groups for compact mode selection. Store selection with `aria-pressed` or `data-ih-active`.

<div class="ih-toggle-group" aria-label="Density">
  <button class="ih-toggle" aria-pressed="true">Comfort</button>
  <button class="ih-toggle" aria-pressed="false">Compact</button>
</div>

The current JS bundle does not manage toggle state yet; update `aria-pressed` in your application logic.

Forms

Autocomplete

Autocomplete is a text input enhanced with a filtered option list. The JS plugin wires focus, filtering, arrow-key navigation, Enter selection, click selection, and Escape close behavior.

Component
  • Button
  • Table
  • Modal
  • Autocomplete
  • Status table
Type to filter, use arrow keys to move, then press Enter or click an option.

    <ul class="ih-autocomplete__list" data-ih-autocomplete-list>
      <li class="ih-autocomplete__option" data-value="Button">Button</li>

When to use

Use autocomplete for option picking, especially when users know what they are looking for.

Keyboard

Arrow keys move through visible options, Enter chooses the active option, and Escape closes the list.

Feedback

Notifications

Alerts communicate brief, non-intrusive state changes. They use severity icons, titles, messages, and optional right-aligned actions.

✓
Build passed

The release artifacts were generated from source.

!
Deployment blocked

One required check is still failing.

i
